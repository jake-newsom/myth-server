Clone cards between databases

Copies characters and variants from the source profile into the destination (merge by character name and variant rarity). Maps sets by name and special abilities by their id slug. Add DATABASE_URL_LOCAL / DATABASE_URL_PROD (etc.) in .env to enable multiple targets. 


              

            

            

              
Patch ZIP (new variants)

              

                Creating or editing a variant with a local image_url queues that file for
                the next graphics patch. Paths inside the ZIP match the client layout (e.g.
                cards/…). Then register the file in
                src/api/routes/assetPatch.routes.ts so clients download it.
