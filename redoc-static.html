Matchmaking

Matchmaking and PvP setup

Join the matchmaking queue

Adds the player to the matchmaking queue with their selected deck. If another player is already waiting, they will be matched immediately and a game will be created.

Authorizations:
bearerAuth
Request Body schema: application/json
required
deckId

Get current matchmaking status

Checks the player's current matchmaking status. Returns whether they are idle, in queue, or have been matched. If in queue, includes wait time and position information. If matched, includes game ID and opponent information.

Authorizations:
bearerAuth

Responses

/api/matchmaking/status

Response samples

  • 200
Content type
application/json
Example
{
  • "status": "idle",
  • "message": "Not in queue or any active match."
}

Leave the matchmaking queue

Removes the player from the matchmaking queue. Can only be used if the player is currently in the queue and not yet matched.

bearerAuth

Responses

  • 400

Retrieve a list of static card data with optional filtering. Card items will not include user-specific fields like user_card_instance_id, level, xp, or power_enhancements.

query Parameters
page
integer >= 1
Default: 1

Page number for pagination

limit
integer [ 1 .. 100 ]
Default: 20

Number of items per page

rarity
string
Enum: "common" "uncommon" "rare" "epic" "legendary"

Filter cards by rarity

name
string

Filter cards by name (partial match)

tag
string

Filter cards by tag

ids
string

Comma-separated list of card IDs to retrieve (e.g. "id1,id2,id3")
